# Zenith Manifesto

Zenith is a universal SaaS launch-and-convert system.

## Pillar 1: The 100/100 Rule

Every Zenith template must be capable of hitting top Lighthouse outcomes when content and assets are production-ready.

### What this means in practice

- Minimal JavaScript and no heavy runtime dependencies
- Semantic HTML structure for crawlability and accessibility
- Performance-conscious CSS and asset usage
- Responsive behavior tested on real mobile breakpoints

## Pillar 2: The Two-Click Rebrand

Brand identity changes should be fast and low-risk.

### Zenith implementation

- Centralized branding tokens in `src/styles/tokens.css`
- Fonts, colors, radii, spacing, and container width all tokenized
- Niche presets using body classes (AI, Finance, DevTools, Studio)

## Pillar 3: Zero Dependency

No required third-party plugin stack.

### Zenith implementation

- Native HTML, CSS, JavaScript only
- No required slider plugin, animation plugin, or UI framework dependency
- Lower maintenance risk and better long-term durability

## Product Promise

Zenith removes friction in three ways:

1. Saves time with reusable conversion sections.
2. Removes confusion with built-in copy formulas and docs.
3. Improves outcomes with conversion-first structure.
